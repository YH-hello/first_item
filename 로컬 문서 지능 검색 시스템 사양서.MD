[사양서] 로컬 문서 지능 검색 시스템 (Local Doc-Intelligence Search)
1. 개요
 - 목적: 외장하드 내 TB 단위 문서(PPT, Word, PDF, Excel, HWP)를 자연어로 검색하고, 결과 클릭 시 해당 파일의 특정 페이지로 자동 이동하는 고도화된 검색 환경 구축.
 - 핵심 가치: 로컬 우선, 딥 링크 기반 작업 효율화, 대용량 계층형 인덱싱.

2. 시스템 아키텍처
 - 데이터 추출 시 텍스트와 함께 '페이지 번호'를 고유 메타데이터(Metadata)로 저장하여 이동 자동화의 근거로 활용합니다.

2.1 계층형 인덱싱 구조
 - L1: 메타데이터 인덱스 (SQLite): 파일 경로, 해시, 수정일 관리.
 - L2/L3: 벡터 인덱스 (Qdrant): 텍스트 벡터 및 이동 자동화용 페이로드(Page, Sheet, Slide ID) 저장.

3. 포맷별 페이지 정보 추출 상세

| 포맷 | 사용 라이브러리 | 페이지 정보 추출 방법 | 자동화 연동 매개변수 (Deep Link) |
| :--- | :--- | :--- | :--- |
| **PowerPoint (PPTX)** | `python-pptx` | 슬라이드 객체 인덱스(1, 2, 3...) 추출 및 슬라이드 노트 매핑 | Slide Index 또는 Slide ID |
| **Word (DOCX)** | `python-docx` | XML 내 `lastRenderedPageBreak` 태그 및 섹션 정보로 추정 페이지 계산 | Page Number |
| **PDF** | `PyMuPDF` (fitz) | `page.number` 속성을 이용한 물리적 페이지 인덱스(0-based) 추출 | `/A "page=N"` CLI 파라미터 |
| **Excel (XLSX)** | `openpyxl`, `pandas` | 시트명(Sheet Name)과 행/열(Row/Col) 주소 기록 | 시트 이름 + 셀 주소 (예: 'Sheet1'!A10) |
| **한글 (HWP)** | `pyhwpx`, `olefile` | 문서 내 제어 문자 및 페이지 구분선 인식을 통한 페이지 번호 추출 | 한글 오토메이션 API 전용 인덱스 |

---
**비고:**
1. **정확도:** PDF와 PPT는 페이지 구분이 명확하나, Word와 HWP는 뷰어 설정(글꼴, 여백)에 따라 페이지가 달라질 수 있으므로 추출 시점의 레이아웃 정보를 최대한 보존해야 합니다.
2. **자동화:** MS Office 제품군은 Windows 환경에서 `win32com` 라이브러리를 통해 강력한 제어가 가능합니다.

4. 데이터 등록 및 인덱싱 프로세스
 - 추출 단계: 파일별 라이브러리를 통해 텍스트와 페이지 매핑 정보를 1:1로 추출.
 - 저장 단계: 벡터 DB의 payload 영역에 { "page_link_cmd": "..." }와 같은 실행 명령 형식을 미리 구성하여 저장.

5. 검색 및 사용자 인터페이스(UI)
 - 검색 방식: Semantic + Keyword Hybrid Search.
 - 결과 표시 (Result Display):
 -- 파일명 및 경로: 로컬 절대 경로 표시.
 -- 페이지 위치: "N페이지" 또는 "N번째 슬라이드" 명시.
 -- 페이지 이동 자동화 (Page-Direct Open):
 --- 검색 결과의 페이지 번호 또는 '열기' 버튼 클릭 시, 해당 소프트웨어를 실행하여 해당 위치로 즉시 점프.
 -- 구현 매커니즘:
 --- PDF: 전용 뷰어 명령행 인자 활용 (예: Acrobat.exe /A "page=5" file.pdf).
 --- Office: Python win32com 라이브러리를 활용한 애플리케이션 원격 제어(Automation).
 --- HWP: 한글 오토메이션 API(HwpCtrl)를 통한 특정 페이지 이동 명령어 실행.

6. 기술 스택 (Technical Stack)
 - Orchestration: LlamaIndex
 - Vector Store: Qdrant (Payload에 실행 명령 스트링 저장)
 - Automation: pywin32 (Windows COM 객체 제어용), subprocess (CLI 명령어 실행)
 - UI: Streamlit (결과 리스트의 클릭 이벤트를 Python 스크립트와 바인딩)
